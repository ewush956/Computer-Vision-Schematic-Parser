# Reconstructor — Status & Next Steps

Snapshot of `SchematicReconstructor` in
[`schematics/schematic_reconstructor.py`](../schematics/schematic_reconstructor.py).

## 1. What is implemented

### 1.1 Data model — `schematics/schematic.py`
Complete. Dataclasses for `BoundingBox`, `Label`, `Component`, `Line`,
`Schematic`. `Line` now carries `status` ("connected" / "dangling" / "orphan")
and a `polyline: list[tuple[int,int]]` field storing the full traced wire path.

### 1.2 Symbol PNGs — `assets/symbols/`
PNG exports of ~43 schematic symbols at 192 DPI with transparent backgrounds.
Filename matches the YOLO class name exactly (e.g. `resistor.png`,
`capacitor.unpolarized.png`). Generated by `extract_symbols.py` from
`assets/Symbols.svg`. Missing classes fall back to a coloured rectangle.

### 1.4 visualize_schematic (standalone function)
A standalone `visualize_schematic(schematic, output_path)` exists at the
bottom of the file and is called from `parser.py`. It does basic rendering
with raw OpenCV calls. **This should be replaced** with a call through the
`SchematicReconstructor` methods once those are implemented.

## 2. What needs implementing

### 2.1 Fix load_xml (DEPANSHU)
Two attribute name mismatches:
- `component.get("conf")` → should be `component.get("confidence")`
- `root.get("image_path")` → should be `root.get("image")`

### 2.2 Filter stubs (DEPANSHU / AMTOJ)
- `filter_by_confidence` — keep components where `confidence >= threshold`
- `filter_by_class` — keep or exclude components by `class_name`

### 2.4 Text linking (AMTOJ)
- `link_text_to_components` — Euclidean center-to-center distance, assign
  matched text component as `Component.label` on nearest non-text component

### 2.5 Rendering methods (EVAN)
- `render_canvas` — white NumPy BGR array sized to schematic dimensions
- `draw_components` — blit PNG from `assets/symbols/{class_name}.png` resized
  to bounding box; fall back to coloured rectangle if PNG missing; skip `text`
- `draw_lines` — draw only `"connected"` lines using `cv2.polylines` on
  `Line.polyline` points (do not draw center-to-center arrows)
- `annotate_labels` — text label above each bounding box
- `summarise` — `Counter` over `class_name`, excluding `text`
- `export_image` — `cv2.imwrite` with format inferred from extension
- **Rewrite `visualize_schematic`** to use the above methods instead of
  raw OpenCV, so `parser.py` continues to work unchanged

## 3. Full pipeline (for reference)

```python
# parser.py drives this:
schematic_recon = SchematicReconstructor()
schematic = schematic_recon.reconstruct(component_xml, polyLines)
# reconstruct() calls load_xml + connect_components internally

# visualize_schematic should internally do:
canvas = recon.render_canvas(schematic)
recon.draw_components(canvas, schematic)
recon.draw_lines(canvas, schematic)
recon.annotate_labels(canvas, schematic)
recon.export_image(canvas, output_path)
```
